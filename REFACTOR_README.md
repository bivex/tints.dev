# Рефакторинг Demo компонента в UI-компоненты

## Что было сделано

Демо-компонент был разбит на атомарные, переиспользуемые UI-компоненты для демонстрации влияния стилей:

### Новые компоненты

1. **`ToggleSwitch`** - Переключатель темного/светлого режима
2. **`FeatureCard`** - Карточка с характеристиками (Uptime, Funding, Features)
3. **`CallToActionSection`** - Секция с кнопками действий
4. **`Rating`** - Компонент рейтинга с звездами
5. **`Tag`** - Тег с иконкой (например, "modern-analogous-1")

### Преимущества рефакторинга

- **Атомарность**: Каждый компонент отвечает за одну функцию
- **Переиспользуемость**: Компоненты можно использовать в других частях приложения
- **Демонстрация стилей**: Легко видеть, как CSS-классы влияют на каждый элемент
- **Удобство поддержки**: Изменения в одном компоненте не затрагивают другие
- **Удаляемость**: Каждый компонент можно легко удалить или заменить

### Как это влияет на стили

Каждый компонент использует Tailwind CSS классы, которые реагируют на:
- **Цветовую палитру** (`first-*` классы)
- **Тему** (`dark:` префиксы)
- **Responsive дизайн** (`md:`, `lg:` префиксы)

### Тестирование

Приложение успешно компилируется и запускается. Все компоненты рендерятся корректно в демо.
